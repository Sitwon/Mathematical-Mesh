LogEntry Structures

Structure: DataItem

TransactionID :
String [0..1]

Unique Transaction ID. Text format ensures that the ID is easily visible in binary formats etc.

PrimaryKey :
String [0..1]

Unique primary key.

PriorTransactionID :
String [0..1]

Specifies the immediately prior transaction that affected this identifier

Action :
String [0..1]

Type of transaction, valid values include: Delete (the key is removed) New (the key is created) Modify (the data associated with the key is modified) Drop (the key is no longer active and will not be modified again)

Added :
DateTime [0..1]

Time at which the item was added to the log

Data :
Binary [0..1]

Binary data.

Text :
String [0..1]

Text data.

Pending :
Boolean [0..1]

If true, transaction is pending and will not be final until a commit transaction is posted.

Commit :
Boolean [0..1]

If true, all pending transactions with the specified TransactionID are committed and cannot roll back.

Rollback :
Boolean [0..1]

If true, all pending transactions with the specified TransactionID are aborted and ignored.

Type :
String [0..1]

Content :

Comment :
String [0..1]

Digest :
String [0..1]

Digest Algorithm

LastDigest :
Binary [0..1]

Final value of last log

Delta :
Delta [0..1]

If populated, this is a delta log.

Structure: Delta

Parent :
String [0..1]

Filename of master log this is a delta to

Previous :
String [0..1]

Filename of previous delta log
