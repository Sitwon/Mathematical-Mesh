MatheMatical Mesh: IANA Registrations

The MatheMatical Mesh requires the following registrations:

  • SRV Prefix for portal discovery: _mmmi.
  • SRV Prefix for intermesh discovery
The relevant authorities are:
  • RFC 6335

SRV Template

Service Name (REQUIRED):           mmm
Transport Protocol(s) (REQUIRED):  TCP, UDP
Assignee (REQUIRED)                Phillip Hallam-Baker 
Contact (REQUIRED)                 Phillip Hallam-Baker 
Description (REQUIRED)

Access protocol for the Mathe-Matical Mesh. The Mesh is a PKI that allows
a personal user to share credentials and application configuration data
between devices in an entirely seamless fashion.
 

Reference (REQUIRED)

The protocol does not make use of IP Broadcast, Multicast or Anycast.

For the time being we would like the application to be considered under the 
'proprietary' rules. The reference code for the protocol is generated from a
machine readable specification. We anticipate making rapid changes to the 
specification during development and would prefer to release an initial Internet 
Draft that reflects experience from alpha testing. 


Port Number (OPTIONAL)            None

Service Code (REQUIRED for DCCP only)   None

Known Unauthorized Uses (OPTIONAL)

None.

Assignment Notes (OPTIONAL)

At this point we are only seeking pre-assignment of the code point _mmm._tcp so
that we can use SRV based discovery in the alpha deployment trial.
